# Contributing to VCARE

Thank you for your interest in contributing to VCARE! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature/fix
4. Make your changes
5. Test thoroughly
6. Submit a pull request

## Development Guidelines

### Code Style

- Use semantic HTML5 elements
- Follow Tailwind CSS utility-first approach
- Write clean, readable JavaScript
- Add comments for complex logic
- Keep functions small and focused

### Commit Messages

Use clear, descriptive commit messages:

```
✨ Add new feature description
🐛 Fix bug description
📚 Update documentation
🎨 Improve styling
♻️ Refactor code
```

### Testing Before Submission

- [ ] Test in multiple browsers (Chrome, Firefox, Safari, Edge)
- [ ] Test responsive design (mobile, tablet, desktop)
- [ ] Test both language modes (English & Gujarati)
- [ ] Verify all form validations work
- [ ] Check for console errors

## Reporting Issues

When reporting bugs, please include:

- Clear description of the issue
- Steps to reproduce
- Expected behavior vs actual behavior
- Browser and OS version
- Screenshots if applicable

## Feature Requests

When suggesting enhancements:

- Explain the use case
- Describe the proposed solution
- Consider alternative approaches
- Include mockups/examples if helpful

## Code Review Process

All submissions will be reviewed for:

- Code quality and consistency
- Functionality and correctness
- Performance impact
- Security considerations
- Documentation completeness

## Questions?

Feel free to open an issue for questions or discussions.
